# Руководство Проекта: Crypto Exchange Simulator

## 1. Функционал Проекта

Этот проект представляет собой симулятор криптобиржи, который использует реальные рыночные данные с Binance, но симулирует исполнение ордеров и баланс пользователя.

### Основные возможности:

1.  **Реальные Рыночные Данные (Binance)**
    *   **График**: Профессиональный график (TradingView Lightweight Charts) с поддержкой свечей, линий, объемов и переключением таймфреймов (1м, 5м, 1ч, 4ч, 1д).
    *   **Стакан (Order Book)**: Визуализация глубины рынка в реальном времени (Bids/Asks) с цветовой индикацией.
    *   **Лента Сделок (Trade Tape)**: Список последних сделок на рынке в реальном времени.
    *   **Тикер**: Обновление цены, изменения за 24ч, High/Low/Volume в шапке.

2.  **Торговый Интерфейс**
    *   Симулированная отправка ордеров (Limit/Market).
    *   Расчет доступного баланса.
    *   История ордеров (в разработке).

3.  **Пользовательская Система**
    *   **Регистрация/Вход**: JWT аутентификация.
    *   **Кошелек**: Просмотр балансов (USDT, BTC, ETH и т.д.).
    *   **Симуляция Депозита**: Начальный баланс 10,000 USDT при регистрации.

4.  **Дизайн и UX**
    *   **Премиум-дизайн**: Темная тема, glassmorphism, анимации.
    *   **Адаптивность**: Полностью работает на мобильных и десктопах.
    *   **Мультиязычность**: Полная поддержка Русского (RU) и Английского (EN) языков.

## 2. Инструкция по Деплою (Развертыванию)

Проект состоит из **Backend** (Django) и **Frontend** (React). Мы будем использовать **Render** для бэкенда и **Vercel** для фронтенда.

### Шаг 1: Подготовка
Убедитесь, что ваш код загружен в репозиторий GitHub.

### Шаг 2: Деплой Бэкенда на Render.com

1.  Зарегистрируйтесь на [dashboard.render.com](https://dashboard.render.com/).
2.  Нажмите **New +** -> **Web Service**.
3.  Подключите ваш GitHub репозиторий.
4.  Заполните настройки:
    *   **Name**: `crypto-exchange-backend`
    *   **Root Directory**: `exchange-sim/backend` (Важно!)
    *   **Runtime**: `Python 3`
    *   **Build Command**: `./build.sh`
    *   **Start Command**: `gunicorn config.wsgi`
5.  Выберите **Free** план.
6.  Добавьте **Environment Variables** (Переменные окружения):
    *   `PYTHON_VERSION`: `3.10.0`
    *   `SECRET_KEY`: `(придумайте любой сложный пароль)`
    *   `DEBUG`: `0`
    *   `ALLOWED_HOSTS`: `*`
    *   `DATABASE_URL`: (добавим позже)
    *   `REDIS_URL`: (добавим позже)
    *   `CORS_ALLOWED_ORIGINS`: `https://ваш-фронтенд.vercel.app` (добавим после деплоя фронтенда)

7.  **Создание Базы Данных (PostgreSQL)**:
    *   В Render Dashboard нажмите **New +** -> **PostgreSQL**.
    *   Создайте базу, скопируйте `Internal Database URL`.
    *   Вернитесь в Web Service -> Environment -> Добавьте `DATABASE_URL`.

8.  **Создание Redis**:
    *   В Render Dashboard нажмите **New +** -> **Redis**.
    *   Создайте Redis, скопируйте `Internal Redis URL`.
    *   Вернитесь в Web Service -> Environment -> Добавьте `REDIS_URL`.

9.  Нажмите **Create Web Service**. Ждите окончания сборки.

### Шаг 3: Деплой Фронтенда на Vercel.com

1.  Зарегистрируйтесь на [vercel.com](https://vercel.com/).
2.  Нажмите **Add New** -> **Project**.
3.  Импортируйте ваш репозиторий.
4.  Настройки проекта:
    *   **Framework Preset**: `Vite`
    *   **Root Directory**: `exchange-sim/frontend` (Важно!)
5.  **Environment Variables**:
    *   `VITE_API_BASE_URL`: `https://ваш-бэкенд.onrender.com` (скопируйте ссылку из Render)
    *   **Важно**: Не ставьте слэш `/` в конце ссылки.
6.  Нажмите **Deploy**.

### Шаг 4: Финальная Настройка

1.  Когда Vercel закончит, скопируйте ссылку на ваш сайт (например, `https://crypto-sim.vercel.app`).
2.  Вернитесь в **Render Dashboard** -> Web Service -> Environment.
3.  Обновите переменную `CORS_ALLOWED_ORIGINS`: вставьте ссылку на ваш Vercel сайт.
4.  Перезапустите backend (Manual Deploy -> Clear Build Cache & Deploy).

**Готово!** Теперь ваш симулятор биржи доступен по ссылке Vercel.
